Methodology & sources

How this map was built

Every line on this map is a real administrative boundary published by New York City. Here is exactly where each one comes from and what was done to it.

What it shows

New York City is carved up by dozens of agencies into overlapping districts that rarely line up with one another. A single address can sit inside a community board, a City Council district, a police precinct, a fire battalion, a school district, a sanitation district, a health area, three different legislative districts and more — all at once. This tool lets you switch any combination of those boundaries on and off and see where they agree and where they cut across each other.

Where the boundaries come from

All 21 layers are official GeoJSON files from NYC Open Data, the great majority published by the Department of City Planning (the others by the agencies noted below). Land layers are the versions clipped to the shoreline where the city provides them. Each dataset ID below links to its source page.

What was done to the data

  • Geometry was simplified. Boundary outlines were thinned with mapshaper (Visvalingam, 5–20% of vertices kept depending on how many shapes a layer has) so that thousands of polygons can load in the browser in under a second. Simplification keeps shapes and their topology but shifts edges by up to a few meters. Do not read exact borders off this map.
  • Labels were generated programmatically. Each area's label (e.g. Precinct 1, Manhattan CB 7, Assembly District 66) is built from the numeric code in the source file. Community-district codes above 18 are "joint interest areas" — parks and airports that belong to no community board — and are labeled as such.
  • Historic districts are filtered to boundaries the Landmarks Preservation Commission marks as current; calendared and superseded study areas are dropped.
  • Nothing was added. No boundary is drawn that isn't in a source file, and no attribute values were invented.
This is a reference and comparison tool, not a system of record. For an authoritative answer to "which district is this address in," use the source agency or the city's official lookups — not a simplified map.

Rebuilding it

The whole pipeline is one script, scripts/build.py: it downloads each dataset's GeoJSON export, normalizes one label per feature, and writes a simplified file plus a manifest.json that drives the map's toggle panel. Layer definitions live in scripts/layers.json. Re-running the script picks up the latest published boundaries.
